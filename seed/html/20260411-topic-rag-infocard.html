Topic-RAG

如何让 AI 成为真正的研究伙伴
困境

研究生问 AI 一个问题，AI 给了一个答案。第二天问一个类似的问题，AI 给了一个看起来差不多但细节不同的答案。第三个月回头看，这三个月积累了几百次问答，没有形成任何东西——换一台机器问同样的问题，AI 什么都不记得。

⚡
我们用 AI 做研究，但 AI 记不住我们在研究什么。它是问答工具，不是研究伙伴。
链路

Semantic Scholar 搜集论文，NotebookLM 建立课题 RAG，Claude Code 驱动主题研究和写作。这条链路本身不新鲜——但把它「按主题封装」这件事，让它变得不一样。

Step 01
搜集
Semantic Scholar 按主题领域抓取论文，建立课题文献库
Step 02
沉淀
NotebookLM 构建课题专属 RAG，形成持续生长的知识库
Step 03
研究
Claude Code 围绕课题做专题研究与写作，调用积累而非从零开始
不是问答，是积累。一个研究课题对应一个 RAG 库，一个 RAG 库对应一个持续工作的 AI。
本质

「按主题封装」这四个字才是关键。不是「用 RAG」，是「为每个研究课题建立专属的知识飞轮」。

Core Insight
当你建立一个 Topic-RAG 库，你是在训练一个了解你这个课题的 AI——它知道你读了哪些论文，知道你的研究方向，知道你踩过哪些坑。伙伴记得你说过什么，工具不记得。
飞轮

你读论文，材料进入 NotebookLM。你产生想法，用 Claude Code 做验证。你得到结果，结果反馈进 RAG 库。下次研究同一个课题，AI 从你的积累出发而不是从空白开始。转得越久，这个 AI 越懂你的课题。

⏳
这件事没有直接回报——不像问 AI 一个问题立刻得到反馈。但它的回报是指数级的。三个月后你会拥有一份积累，会在每一次新的研究中放大你的效率。
