## 背景

信息爆炸的时代，碎片涌入的速度远超人脑处理的上限。大多数知识管理系统的失败，不在于存储不够，而在于输入失控——没有区分度，所有信息被平等对待，垃圾和珍珠一起淹没在同一条河流里。

真正有效的知识系统，必须是一道**过滤网**，而不是一个储物间。

本文记录我所构建的自动化信息流架构，核心是三个组件：**Hermes Agent**（处理引擎）、**Obsidian**（数据持久层）、**Hub UI**（可视化层）。


## 两条输入路径

### 路径一：主动对话输入

这是权重最高的输入通道。

当我在手机或电脑上有了一段确定的、高价值的认知——读了一篇深度好文、产生了一个重要的想法、记录了一段值得沉淀的思考——我会直接通过对话框发给 Hermes。

```text
输入 → Hermes 即时解析 → 与记忆系统建联 → 回写 Obsidian + 触发 Hub 更新
```

这个路径没有中间商。确定性知识直接进入系统，不需要沙盒评估。

关键在于**即时结构化**。Hermes 不会只是存档，它会主动将新信息与已有的记忆网络进行关联——这个新输入补充了哪个既有主题？强化了什么判断？纠正了什么偏见？这些内化动作在消息发出的那一刻同步完成。


### 路径二：Obsidian 目录触发

这是日常无感收集的通道。

我在多端浏览时，通过剪藏工具将碎片信息推送到 Obsidian 的指定目录，由 Hermes 定时扫描处理。这个路径的关键不是实时，而是**自动路由**。

核心逻辑基于目录感知：

**01 - Learn 目录（核心内化区）**

这个目录里的内容被视为确定性知识。剪藏进来后，Hermes 执行无条件吸收——跳过价值评估，直接提取、内化，并融汇到现有的知识网络中。

这是白名单机制。进入这个目录的信息，享有与路径一同等的信任级别。

**00 - Inbox 目录（待定沙盒）**

这是大多数碎片信息的着陆点。信息权重未知，不直接内化。Hermes 在这里扮演**知识海关**的角色：

- 扫描评估内容的价值
- 主动匹配既有主题（Themes）
- 命中则放行，执行自动化归档
- 无关联则滞留，不做任何拓展

最重要的约束：**禁止越权创建新主题**。系统不会因为一条无关联的碎片就自行生成一个新的topic方向。主题是长期积累的结果，不是一时碎片灵感的产物。


## 路由策略的精确表述

```text
IF 目录 == "01 - Learn":
    → 无条件内化，更新记忆网络

IF 目录 == "00 - Inbox":
    IF 内容命中既有主题:
        → 提取 + 归档
    ELSE:
        → 滞留，禁止创建新主题

IF 处理完毕:
    → 移动至 XX - Archived（只读存档）
```


## 为什么是 Obsidian

选择 Obsidian 作为持久层，有三个原因。

第一，**本地优先**。文件在本地磁盘，不依赖任何云服务，不会有隐私风险，也不存在服务下线的问题。

第二，**目录即语义**。通过目录名称表达信息权重——这是人可以直接理解和控制的，不需要复杂的配置。

第三，**双链积累**。随着使用时间的增长，Obsidian 中的笔记之间会自然形成网络，这个网络本身就是思维的外化。


## 系统边界

这个架构解决的是**信息输入**的问题，不是信息输出的问题。

输出（写作、分享、行动）有另一套逻辑，不在本文讨论范围内。

系统的设计哲学是：**接收时严格，消化时深入，存储时克制**。宁可漏掉一些，不要塞进来垃圾。


## 总结

两条输入路径，一个处理引擎，三层目录路由，构成了一个自运转的信息入口。

人只需要做一件事：**在正确的时机，把正确的东西放到正确的目录里**。

其余的，Hermes 来处理。
